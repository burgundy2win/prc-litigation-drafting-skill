## 1. 审查和交付原则

- 围绕诉讼请求、抗辩要件、举证责任和争议焦点确定是否入证、如何分组、排列顺序和证明目的。
- 将用户认为有用以及律师判断具有证明、反驳、解释、补强或程序价值的材料纳入审查。对重复、无关、来源不明或可能产生反作用的材料，说明后决定是否提交。
- 默认交付：证据目录Word、全部证据合并且连续编页的单一PDF；必要时附情况说明、代付说明、电子数据提取说明、金额计算表或律师核查提示。
- 目录与证据内容PDF分开交付，目录页不计入证据材料页码。除用户明确要求外，不把各项证据拆成多个最终内容文件。

## 2. 目录Word版式

- A4纵向；页边距上、下各2.54cm，左、右各2.5cm；中文统一使用仿宋_GB2312、黑色。
- 封面标题`证据材料`：22磅、加粗、居中。
- 目录标题`证据目录`：18磅、加粗、居中，固定值25磅，段后6磅。
- 案件基本信息：12磅、小四号、固定值20磅；字段名加粗，内容不加粗。
- 目录表和登记表正文12磅；表头12磅、加粗、水平及垂直居中。长文本左对齐，序号和页码居中。
- 表格单倍行距，段前段后0；单元格上下内边距0.15cm、左右0.18cm。不得用空格或回车撑高。
- 表格跨页重复标题行；同一证据的编号、名称和页码不得拆页。证明目的长行可跨页，但不得截断或越界。
- 目录自身页码从封面后的正文起算，封面不显示；页脚居中、阿拉伯数字、仿宋_GB2312、10.5磅。不得与证据PDF页码混用。

## 3. 封面及案件字段

封面至少包含：

- 主标题：`证据材料`。
- 案由：法院立案案由或拟主张的规范案由。
- 案号：完整案号；未取得时写`案号：待定`或留空。
- 提交人：当事人全称及诉讼地位，如`被告：×××`。
- 诉讼阶段：一审、二审、再审、执行异议等。
- 提交日期：与实际提交日期一致，未确定时留空。

统一使用“字段名：字段值”，不得用文件夹名称或非规范简称代替。

## 4. 编号和分组

- 围绕同一待证事实形成证据链的材料编为一组；证明目的或法律关系不同的原则上分组。
- 组号使用`第一组证据、第二组证据……`；组内使用`1-1、1-2……`、`2-1、2-2……`。
- 简单案件可用`证据1、证据2……`，但同一目录不得混用编号体系。
- 合同及其附件可以作为一项；合同、履行记录、付款凭证和发票等性质不同的材料分别列为子项。
- 补充提交原则上续接原编号和页码；必须插入时使用`1-3补1`等可追溯编号，避免与已提交版本混淆。

## 5. 目录总表

总表固定设置四栏：

| 栏目 | 填写标准 |
| --- | --- |
| 证据编号 | 填组号；不分组时填单项编号。须与登记表、PDF书签和答辩状脚注一致。 |
| 证据名称及材料构成 | 先写概括性组名，再按子编号列正式名称；名称包含足以识别的主体、事项或日期。禁止“聊天记录若干”“转账记录一份”等模糊名称。 |
| 证明目的 | 以`证明：`起句，分项写明主体、时间、行为、金额、法律关系或程序事项，并说明组内材料的印证关系。 |
| 页码 | 单页写`第X页`，连续多页写`第X—Y页`；不连续材料逐项列明，不得只写总页数。 |

名称标准格式：

`第一组证据：关于××事实的证据`

`1-1《××合同》（签订日期：×年×月×日）`

`1-2××银行转账凭证（付款人××，收款人××，转账日期：×年×月×日）`

## 6. 民事证据登记表

每组设置一张登记表，固定字段如下：

| 字段 | 填写标准 |
| --- | --- |
| 证据编号 | 与总表一致；组内材料另列子编号。 |
| 证据名称 | 使用正式名称；无标题时按“主体＋事项＋载体＋日期”客观命名。 |
| 证据种类 | 选择书证、物证、视听资料、电子数据、证人证言、当事人陈述、鉴定意见、勘验笔录；复合证据分别标明。 |
| 证据来源 | 写明出具、形成、提供或调取主体及取得方式；电子数据写明平台、账号或设备。不得仅写“客户提供”。 |
| 形成时间 | 填写形成、签署、发送、交易或出具日期；时间跨度写起止日期；无法确定写`形成时间待核`。 |
| 原件及载体情况 | 写明原件、复印件、扫描件、打印件或电子数据，原件持有人及能否当庭核验；电子数据写明原始载体是否保留。 |
| 证据内容摘要 | 客观概括主体、行为、时间、金额、关键条款或沟通内容，不作证明力评价。 |
| 证明目的 | 对应争点或法律要件写待证事实，不得仅写“证明案件事实”“证明对方违法”。 |
| 页码范围 | 与最终PDF连续页码一致，各子项分别列起止页。 |
| 核验及风险提示 | 写真实性、合法性、关联性、完整性、名称差异、签章缺失、第三方代付、截取不完整、原件缺失或需申请调取等问题。无明显问题写`已作形式核验，证明力由法院结合全案认定`。 |

## 7. 证明目的

- 使用“证据事实＋要件意义”：先写证据直接反映的事实，再写其与请求、抗辩或程序事项的关系。
- 推荐格式：`证明：1. ××与××于×年×月×日订立××合同，约定……；2. ××已于×年×月×日履行……，金额为……；3. 上述材料相互印证……。`
- 证据不能直接证明最终结论时，只写能够支持的中间事实；不用“充分证明”“完全证明”“唯一证明”等夸张措辞。
- 复印件、片段截图、单方制作或真实性待核材料，使用“反映”“初步证明”“与其他证据共同证明”，并在风险提示中说明限制。
- 多项材料共同证明同一事实时，组层面写综合目的，各子项写具体贡献，避免复制相同长句。

## 8. 材料排序及分类整理

- PDF顺序、目录、登记表、答辩状脚注和书签必须一致。
- 组内一般依次排列：基础法律关系文件或主文书；形成和变更材料；履行材料；付款或收款凭证；发票及结算；催告、通知或沟通；情况说明和辅助核验材料。
- 同类型材料按时间从早到晚；因争点需要倒序时注明。重复件删除；不同版本有证明价值时分别命名日期和差异。
- 身份证明、授权委托和送达地址确认等程序材料原则上不作实体证据；涉及主体资格、授权或程序争点时除外。
- 合同、函件、票据和裁判文书保留首页、正文、附件、签章页及骑缝章，不只截取有利段落。
- 银行流水和支付凭证保留账户名、必要账号信息、时间、金额、对方户名、摘要或流水号；无关隐私可适度遮盖，但不得妨碍核验。
- 聊天、短信和邮件保留账号主体、对方账号、时间、上下文和连续性；不得将关键内容脱离上下文。必要时附提取过程说明。
- 录音录像提交可播放原文件，并制作载体清单、形成方式说明和标注时间点的文字整理稿；文字稿不替代原文件。
- 网页和平台数据记录网址、平台、账号和提取时间；易变内容优先以完整页面、下载文件、公证或可信时间戳固定。
- 图片和扫描件保持方向正确、文字可读、边缘完整，不拉伸、不改变原意，不裁切签章、日期、金额等关键字段。
- 外文材料保留原文并按提交要求附中文译本，原文与译文分别标识并相邻排列。

## 9. PDF合并、书签和编页

- 全部证据合并为一个PDF；目录不并入证据内容PDF，除非用户或法院明确要求一体装订。
- 第一页固定编为`1`，之后连续编号，不因分组、横竖页、原文件页码或附件重新起算。
- 页码实际写入每页页面内容，不能只设置PDF逻辑页码。
- 新编页码为黑色阿拉伯数字、12磅，置于阅读方向右下角，距右边约1.0cm、距下边约0.8cm。横向页面按阅读方向处理。
- 保留原材料自带页码，不涂改；目录定位以新增右下角连续页码为准。
- 不得遮挡正文、签名、印章、金额、二维码或原页码。右下角有重要内容时，优先增加不裁切原文的白色页脚空间；无法增加时在附近空白处最小幅度平移，并保持全卷一致。
- 书签至少设置到每组，命名为`第一组证据｜概括性组名`；材料较多时增加组内单项书签。
- 无意义空白页删除；有核验意义的空白背页、骑缝章页或版式关系予以保留并注明。

## 10. 制作顺序

不得先猜页码再合并，固定顺序为：

1. 确定入证范围、分组、编号和排序。
2. 转换、校正并合并证据。
3. 删除无意义空白页，检查方向、尺寸和清晰度。
4. 写入右下角连续页码并设置书签。
5. 读取最终总页数和每项证据边界页。
6. 回填目录总表、登记表和答辩状脚注。
7. 复核目录、登记表、书签和页面页码。

页码栏：单页`第8页`；连续多页`第8—12页`；不连续页`第8—10页、第15页`。不用`P8-P12`、`8至12`或仅写`5页`。

## 11. OCR和质量核验

- 文本型PDF优先保留并提取文本层；简单扫描件先使用OCRmyPDF/Tesseract添加可搜索文本层。OCR不得改变页面视觉内容。
- 姓名、日期、金额、账号、案号、签名、印章、二维码和流水号必须目视核验，不直接采用未经核验的OCR结果。
- 图片转PDF保持原始比例和足够分辨率；横向材料可保留横向，不强制缩为字迹过小的纵向页。
- 检查缺页、重页、倒置、裁边、黑页、乱码、透明层或签章丢失、文件损坏、页码遮挡和书签错误。
- PDF总页数必须等于最后一个连续页码；逐组检查首尾页。目录、登记表、书签、答辩状脚注和PDF页码必须完全一致。
- Word目录须检查字体替换、表格跨页、重复标题行、文字截断、空白页、页码和打印效果。
- 文件名使用“当事人或案件简称＋材料类型＋版本日期”，例如`张某诉李某民间借贷纠纷-证据材料-20260714.pdf`，不用“新建文档”“最终最终版”。
