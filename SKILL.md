# 民事诉讼文书与证据材料

## 核心原则

- 以律师身份独立审查，不机械照抄用户思路或材料标题。
- 围绕诉讼请求、抗辩要件、举证责任、争议焦点和程序问题组织事实、理由和证据。
- 不补造事实。信息不足但不妨碍继续工作的，在工作稿标注 `【待确认】`，同时列出核查清单；定稿前清除或取得确认。
- 先完成证据材料的最终排序和连续编页，再回填证据目录及答辩状脚注页码。
- 不依赖任何历史案件或特定模板；直接执行本 Skill 的标准化结构、字段和版式。用户提供案件专用模板或法院明确格式时，以当次要求为准。
- 起诉状和鉴定申请书仅复用通行要素、排版和核验逻辑；每案当事人、主体信息、联系方式、请求、事故或履行事实、金额、责任主张、鉴定事项及法院均须从本案材料提取，不得借用其他案件内容。
- 法律文档中的修订、批注和审阅元数据作者统一设为“审阅律师”，不得出现 Codex 或 AI 身份。

## 按任务读取规则

- 仅处理答辩状：完整读取 [references/defense-brief.md](references/defense-brief.md)。
- 处理民事起诉状或鉴定申请书：完整读取 [references/complaint-and-appraisal.md](references/complaint-and-appraisal.md)。
- 处理证据目录、登记表、证据内容或证据 PDF：完整读取 [references/evidence-package.md](references/evidence-package.md)。
- 同时处理多项诉讼文书与证据：读取相应全部参考文件，并统一当事人、请求、事实、证据编号、证明目的和页码。
- 创建或修改 DOCX：同时使用 `documents:documents` Skill，执行其渲染与验证要求。修改现有法律 Word 文档时，实质性修改使用 Track Changes，并用批注说明重要风险和待决事项；用户明确要求清洁定稿时，另行提供无修订痕迹版本。
- 创建、合并、OCR、编页或核验 PDF：同时使用 `pdf:pdf` Skill。文本型 PDF 先提取文本；简单扫描件先本地 OCR；页面渲染用于关键字段和最终版式核验。

## 输出范围和版本状态

- 用户要求“写”“起草”或“按标准流程写”答辩状、起诉状或鉴定申请书而未指定载体时，默认创建正式 DOCX；用户明确要求先看文字、提纲、思路或聊天草稿时，才以内联文本为主。
- 资料完整且无待确认事项时，直接形成清洁定稿。存在可能影响请求、事实、自认、金额或证据对应关系的实质缺口时，交付明确标识的工作稿，保留`【待确认】`并附核查清单，不把工作稿冒充定稿。
- 仅要求答辩状时，不得擅自扩大为制作证据目录或证据PDF。已有最终证据PDF的，脚注填写准确页码；尚无最终证据PDF的，脚注先写证据编号和名称，并用`【证据页码待最终证据卷编定后回填】`标识，不猜测页码，同时在核查清单中提示回填。
- 用户明确要求清洁定稿但仍缺少关键事实或证据页码时，删除内部工作标记前必须以不虚构事实的方式改写；无法安全定稿的，说明缺口并保留为工作稿。

## 标准工作流

### 1. 接收和盘点材料

- 列出全部文件，识别起诉状、答辩材料、合同、履行凭证、付款记录、沟通记录、程序材料和既有模板。
- 优先使用项目自带虚拟环境；处理 OCR/PDF 时，按本机环境配置启用相应工具链。
- 核对当事人、法院、案号、案由、诉讼阶段、提交期限及用户要求的交付范围。

### 2. 建立事实—争点—证据关系

- 拆解对方每项诉讼请求、事实主张和法律依据。
- 识别实体抗辩、程序抗辩、不利自认风险、金额计算问题和证据缺口。
- 将每项待证事实映射至具体证据；区分直接证据、补强证据、反驳证据和仅具线索价值的材料。
- 对真实性来源、原件、主体名称差异、第三方代付、签章、账号主体、聊天连续性等问题提出补强建议。

### 3. 按请求执行

- 答辩状：按参考规则形成明确的答辩请求、分层事实与理由、证据脚注和签章栏。
- 起诉状：按参考规则形成完整当事人信息、明确诉讼请求及诉讼标的暂定金额、事实与理由、管辖法院和签章栏；增加或变更被告时，逐一核验其统一社会信用代码、住所或营业场所、法定代表人或负责人、职务及送达联系方式。
- 鉴定申请书：与起诉状的当事人称谓和主体信息完全一致；明确鉴定事项、目的、范围及与待证事实的关系，不重复或扩大未经材料支持的鉴定项目。
- 证据目录：形成封面、案件基本信息、目录总表和逐组民事证据登记表。
- 证据内容：按目录顺序转换、校正、合并为一个 PDF，设置书签并在每页右下角写入连续页码。
- 综合任务：先定证据范围和顺序，再制作证据 PDF，随后回填目录和答辩状脚注，最后统一核验。

### 4. 质量核验

- 核对姓名或名称、身份信息、案号、案由、法院、日期、金额、签名、印章、账号和流水号。
- 核对起诉请求或答辩请求、鉴定事项、正文论证、证明目的、证据名称、编号、PDF书签及连续页码是否完全一致。
- 检查 DOCX 的字体、标题层级、表格跨页、脚注、修订作者、空白页和打印效果。
- 检查 PDF 的缺页、重页、方向、清晰度、裁边、遮挡、OCR错误、书签和每组首尾页。

## 默认交付

按用户请求交付下列一项或多项：

1. 答辩状、起诉状或鉴定申请书 `.docx`；修改既有文书时默认保留审阅痕迹。
2. 证据目录 `.docx`，含封面、总表和逐组登记表。
3. 按目录顺序合并、设置书签并连续编页的证据材料 `.pdf`。
4. 必要的金额计算表、情况说明、代付说明、电子数据提取说明或律师核查清单。

除用户明确要求外，不将各项证据拆成多个最终内容文件。
